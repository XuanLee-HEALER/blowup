# blowup 布局规范

## 概述

blowup 是一个基于 Tauri 2 + React + Mantine 9 的 macOS 桌面电影工作流工具，覆盖从发现、获取、管理、加工到知识沉淀的完整链路。本文档定义应用的整体布局结构、导航逻辑、各空间内容规划、现有页面迁移方案及实现要点。

---

## 整体结构

应用窗口分为三个纵向区域，从左到右依次为：

1. **图标侧边栏**（固定 48px）
2. **主内容区**（弹性填充）
3. **上下文面板**（固定 320px，按需显示）

三者水平排列，高度为窗口全高。

---

## 窗口与标题栏

Tauri 窗口配置为 `decorations: false` + `titleBarStyle: overlay`，使用系统原生交通灯按钮叠加在自定义内容上方，而非自行绘制关闭/最小化/最大化按钮。

不需要单独渲染一个标题栏横条。整个窗口顶部区域通过 CSS `-webkit-app-region: drag` 设定为可拖拽区域。图标侧边栏和主内容区各自在顶部预留拖拽空间。

交通灯按钮由系统绘制，位于窗口左上角。由于交通灯与图标侧边栏在垂直方向上重叠（侧边栏从窗口顶部开始），侧边栏顶部的第一个空间图标必须向下偏移至少 40px，为交通灯腾出空间。

窗口最小尺寸约束：`minWidth: 900px, minHeight: 600px`，确保三栏布局在最小尺寸下仍然可用（48 + 480 + 320 = 848，留有余量）。

---

## 图标侧边栏

宽度固定 48px，背景色使用比主内容区略深一级的次级背景色（Mantine 的 `--mantine-color-body` 对应的次级色），右侧有 0.5px 的分割线。

### 空间图标

侧边栏内包含三个空间图标，垂直排列在顶部（注意第一个图标距顶部至少 40px，避开交通灯区域），间距 8px，每个图标区域 48x40px，图标本身 20x20px 居中放置。

三个空间：

1. **影片库**（Library）— 本地影片管理、浏览、暗房操作的入口。图标：胶片/网格。
2. **发现**（Discover）— TMDB 搜索、种子搜索、下载任务管理。图标：搜索/指南针。
3. **知识库**（Knowledge）— Wiki 编辑器、知识图谱浏览。图标：书本/节点图。

当前激活的空间图标使用高亮状态（背景色 + 图标变色），未激活的图标使用低对比度的次级文本色。

### 底部

侧边栏底部放置设置图标（齿轮），与空间图标之间用 `flex: 1` 的空白撑开。

不使用任何文字标签。图标建议使用 Tabler Icons（Mantine 生态常用）。

### 快捷键

`Cmd+1` / `Cmd+2` / `Cmd+3` 切换三个空间。点击设置图标打开设置（设置不是一个"空间"，而是一个覆盖视图，见下文设置部分）。

---

## 主内容区

占据侧边栏右侧到上下文面板左侧之间的全部空间，`flex: 1` 弹性填充，设置 `min-width: 480px` 防止过度压缩。

### 工具栏

主内容区顶部是一个 40px 高的工具栏区域（非独立组件，是主内容区内部的顶部空间），用于放置当前空间内的二级导航、搜索输入框、筛选控件、视图切换按钮等。工具栏的具体内容随空间和页面变化。

工具栏区域同样需要部分设置为可拖拽区域（按钮和输入框等交互元素需设置 `-webkit-app-region: no-drag` 排除）。

### 页面内容

工具栏下方是页面内容的滚动区域，`overflow-y: auto`。

空间内的页面跳转使用 React Router 实现前端路由，不暴露 URL 或浏览器导航控件。如需"返回"操作，在工具栏中放置返回箭头按钮。

## 各空间内容规划与现有页面迁移

### 影片库（Library）

**合并原"影片"页面和"暗房"页面。**

现有系统中，影片页面和暗房页面共享完全相同的左侧导航树（按导演分组的影片列表），只是右侧展示不同内容（影片详情 vs 文件操作）。新布局将二者合并为一个空间：主内容区展示影片导航，上下文面板内通过 tab 切换详情和暗房操作。

**主内容区**展示影片列表，支持两种视图模式：

- **列表模式**（默认）：延续现有的按导演分组树形结构。导演名作为分组标题（加粗，右侧显示影片数量），下级是该导演的影片条目（标题 + 年份）。列表现在占据整个主内容区宽度（而非现有的约 200px 窄面板），每个条目可以显示更多信息：时长、分辨率、文件大小、字幕状态图标等。
- **网格模式**：海报缩略图网格，适合浏览。

单击影片条目 -> 右侧上下文面板展开，展示该影片的全部信息。

**工具栏内容**：搜索框（对应现有的"搜索标题或导演..."输入框）、视图切换按钮（列表/网格）、排序方式下拉、更多操作菜单（包含"重建索引"，不再作为独立按钮）。

### 发现（Discover）

**合并原"搜索"页面和"下载"页面。**

原系统中搜索和下载是两个独立的侧边栏入口。新布局中它们作为发现空间内的两个子视图，通过工具栏中的 tab 切换。

**搜索子视图**：

- 搜索输入框集成在工具栏中（对应现有搜索页面的 "电影名称、导演... (回车搜索)" 输入框）。
- 工具栏下方展示筛选标签（年代、类型、评分、按评分排序），对应现有的筛选控件。
- 搜索结果以列表展示。
- 选中某个搜索结果 -> 右侧上下文面板展示该片 TMDB 详情、可用种子列表（来源 YTS/YIFY、文件大小、清晰度）、下载按钮。如果本地已有该片，显示关联状态和跳转到影片库的链接。

**下载队列子视图**：

- 对应现有下载页面的完整内容。
- 每个任务显示：片名、导演、分辨率、状态标签（下载中 / 已完成 / 失败）、进度条（下载中时）。
- 右侧操作按钮保持不变（重新下载、删除）。
- 下载任务选中时上下文面板可选择性展开（如果列表信息已足够则不需要面板）。

**工具栏内容**：子视图切换 tab（搜索 / 下载队列），搜索子视图激活时搜索框集成在工具栏中。

### 知识库（Knowledge）

**合并原"Wiki"页面和"知识图谱"页面。**

原系统中 Wiki 是 master-detail 的列表+编辑器结构，知识图谱是独立的力导向图画布。新布局中它们作为知识库空间内的两个视图模式切换。

**列表视图**（对应原 Wiki 页面）：

- 主内容区展示条目列表。列表现在占据整个主内容区宽度（而非现有的约 200px 窄面板）。每个条目显示标题和更新时间。
- 列表上方有搜索框和"+ 添加"按钮（对应现有的搜索框和"+ 添加"链接）。
- 选中条目 -> 右侧上下文面板展示条目预览和元数据。
- 双击条目或在上下文面板中点击"编辑" -> 主内容区切换为全宽编辑器视图。编辑器内工具栏提供返回按钮回到列表视图。编辑状态下上下文面板可隐藏或展示文档大纲/关联信息。

**图谱视图**（对应原知识图谱页面）：

- 主内容区变为全幅力导向图画布。节点（蓝色圆形）和关系边（带标签的有向连线，如"作品/导演"）的渲染逻辑保持不变。
- 点击节点 -> 右侧上下文面板展示该节点（条目）的详情和关联关系。
- 图谱视图下工具栏可包含缩放控件和布局重置按钮。

**工具栏内容**：条目搜索、新建条目按钮（"+ 添加"）、视图切换（列表/图谱）。

## 上下文面板（右侧面板）

宽度固定 320px，左侧有 0.5px 分割线，背景色与主内容区一致或使用可选的次级背景色。

### 显示/隐藏逻辑

- 主内容区有选中项时显示
- 取消选中时隐藏（点击主内容区空白区域或按 `Esc`）
- 快捷键 `Esc` 关闭面板
- 快捷键 `Cmd+F` 聚焦当前空间的搜索框

### 动画

面板出现和消失使用横向 slide 动画，150-200ms，ease-out 曲线。不使用 `display: none/block` 切换，而是始终渲染面板 DOM，通过 `transform: translateX(320px)` + `opacity` 控制显隐。面板显示时主内容区宽度相应收缩（非 overlay），确保列表内容不被遮挡。

也可使用 Mantine 的 `Transition` 组件实现。

### 响应窗口尺寸

当窗口宽度不足（小于约 850px）时，上下文面板应自动切换为 overlay 模式叠加在主内容区上方，或自动隐藏，而非继续挤压主内容区。

### 各空间的面板内容

**影片库 -- 选中影片时：**

面板内部使用 Mantine Tabs 组件，分为两个 tab：

**详情 tab**（对应现有影片页面右侧内容）：
- 海报缩略图（宽度不超过面板宽度的 40%，约 120px）
- 标题（大字）
- 年份 + 评分（如 "1966 · ★ 8.1"）
- 元数据字段：导演、主演、编剧、摄影、配乐、剪辑、制片（保持现有的"标签: 值"格式，但改为更紧凑的排列）
- 类型标签（如 "Drama"）
- 视频文件区块：文件名（ellipsis 截断）+ 播放按钮 + 删除按钮
- 字幕文件区块：列出所有字幕文件（同样 ellipsis 截断）
- 底部小字：文件路径、添加时间

**暗房 tab**（对应现有暗房页面右侧内容）：
- **视频**区块：文件名 + 播放按钮 + 媒体信息按钮 + 更多操作（三点菜单）。下方显示元信息：格式、大小、时长、码率，以及流信息（如 "#0 video -- h264 988x720 23.98 fps" 和 "#1 audio -- aac 2ch 48000Hz"）。
- **字幕**区块：列出所有字幕文件，每个字幕文件旁有操作按钮（对齐、查看、删除）。对齐后生成的 `.aligned.ass` 文件以缩进子项形式展示在原字幕下方。底部有"+ 搜索字幕"按钮（旁边带语言选择下拉，默认"中文"）和"+ 从视频提取"按钮。
- **音频**区块：列出已提取的音频文件（如 `_audio_0.m4a`），每个有播放按钮和删除按钮。

320px 面板内的空间约束：
- 文件名使用 `text-overflow: ellipsis` 截断，hover 时 tooltip 显示完整名称
- 操作按钮优先使用图标按钮（而非文字按钮）以节省空间
- 媒体信息（格式、编码等）使用 12px 小字、次级文本色

**发现 -- 选中搜索结果时：**

- TMDB 影片详情（海报、标题、年份、评分、简介）
- 可用种子列表（来源 YTS/YIFY、文件大小、清晰度），每个种子旁有下载按钮
- 如果本地影片库已有该片，显示"已在影片库中"状态标签和跳转链接

**知识库 -- 选中条目时：**

- 条目标题和创建/更新时间
- 条目摘要或预览（Markdown 渲染的前几段）
- 关联关系列表（链接到的其他条目，可点击跳转）
- 如果条目关联了本地影片，显示"在影片库中查看"跳转链接
- 快捷操作按钮：编辑、在图谱中定位、删除

## 设置

设置不是一个"空间"（不占用图标侧边栏的空间位置），而是一个覆盖视图。

点击侧边栏底部的设置齿轮图标 -> 主内容区和上下文面板区域被设置页面覆盖（图标侧边栏保持可见，设置图标高亮，点击其他空间图标可退出设置返回该空间）。快捷键 `Cmd+,` 打开设置。

设置页面内容按分组展示，保持现有分组逻辑：

- **TMDB**：API Key
- **OpenSubtitles**：API Key、用户名、密码
- **ASSRT（射手网）**：Token
- **字幕**：默认语言下拉
- **工具路径**：ffmpeg 路径

设置页面使用单栏居中布局，最大宽度 600px，不显示上下文面板。

## 页面迁移对照表

| 原侧边栏入口 | 新归属 | 迁移说明 |
|---|---|---|
| 搜索 | 发现空间 - 搜索子视图 | 搜索输入框移入工具栏，筛选标签保留 |
| Wiki | 知识库空间 - 列表视图 | 条目列表扩展到主内容区全宽，选中条目后右侧面板展示预览 |
| 知识图谱 | 知识库空间 - 图谱视图 | 画布占满主内容区，点击节点右侧面板展示详情 |
| 影片 | 影片库空间 | 导演分组列表扩展到主内容区全宽，选中影片后右侧面板的"详情" tab 展示 |
| 下载 | 发现空间 - 下载队列子视图 | 列表内容不变，作为发现空间内的子视图 tab |
| 暗房 | 影片库空间 - 上下文面板的"暗房" tab | 不再独立页面，合并为影片上下文面板的第二个 tab |
| 设置 | 设置覆盖视图 | 从侧边栏底部齿轮进入，覆盖主区域，Cmd+, 快捷键 |

## 路由结构

```
/library                    -> 影片库（列表/网格视图）
/library/:movieId           -> 影片库，指定影片选中（上下文面板展开）

/discover                   -> 发现 -> 搜索子视图
/discover/downloads         -> 发现 -> 下载队列子视图

/knowledge                  -> 知识库 -> 列表视图
/knowledge/graph            -> 知识库 -> 图谱视图
/knowledge/edit/:entryId    -> 知识库 -> 编辑器视图

/settings                   -> 设置覆盖视图
```

路由变化时，图标侧边栏的激活状态根据路由前缀自动匹配：`/library` -> 影片库高亮，`/discover` -> 发现高亮，`/knowledge` -> 知识库高亮，`/settings` -> 设置齿轮高亮。

## 布局实现

使用 CSS Flexbox：

```
窗口容器: flex, flex-direction: column, height: 100vh
  +-- 内容行: flex: 1, flex-direction: row, overflow: hidden
        +-- 图标侧边栏: width: 48px, flex-shrink: 0
        +-- 主内容区: flex: 1, min-width: 480px, overflow: hidden
        |     +-- 工具栏: height: 40px, flex-shrink: 0
        |     +-- 页面内容: flex: 1, overflow-y: auto
        +-- 上下文面板: width: 320px, flex-shrink: 0 (动画控制)
```

窗口顶部拖拽区域通过在侧边栏顶部空白区域和主内容区工具栏设置 `-webkit-app-region: drag` 实现。所有交互元素需设置 `-webkit-app-region: no-drag` 排除。

## 视觉风格备注

当前阶段使用 Mantine 9 默认主题搭建结构，后续统一定制色彩和排版方案。

关键原则：

- 文字信息优先于视觉装饰
- 列表视图保持紧凑的信息密度
- 海报/图片只在需要时展示（上下文面板中的小海报、网格视图），不做装饰性铺陈
- 整体色调偏冷、克制，避免过度饱和的强调色
- 面板内操作按钮优先使用图标按钮，hover tooltip 说明功能
- 长文件名一律 ellipsis 截断，tooltip 显示完整路径
